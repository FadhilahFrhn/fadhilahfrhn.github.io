Stackoverflow

Back again with the Stackoverflow dataset, different from before, this time I processed the dataset using Python to answer several business questions that required me to create charts from the insights I had obtained.

The first question, I was asked to make three line charts, each diagram representing a group of questions on Stackoverflow, the group consisting of BigQuery, Data-Studio, and Looker questions. The number of questions grouped is based on the month in the year the question was created.

The first step I carried out Exploratory Data Analysis, at this stage I started to get acquainted with the dataset so that I could determine key columns to answer the business questions given, then I carried out Data Cleaning by eliminating duplicate data so as not to interfere with the data aggregation process, finally I processed the aggregated data into a line chart that is easy to read.

Apart from answering this question, I have also succeeded in answering two other questions which are no less interesting, explore them more deeply by pressing the view project button below.

Stackoverflow data in real life

In this project I focused on learning and practicing using the SQL language. The platform I use is Google BigQuery sandbox, the reason is because I want to utilize the public dataset available in Google BigQuery, in this project I use the Stackoverflow dataset. I don't use all the tables, I only use the post_questions, post_answers, users, and badges tables. From the four tables, I try to dig insights through the relationship between one table and another.

I got some interesting insights, including, I managed to present a list of all questions and their answers related to the Python programming language in 2020, then how many questions were made by Indonesians from 2015 to 2020, then in which year questions about BigQuery received the most answers, then calculated the average time span required by the username to get the answer to the question and I made a distribution based on the 10th, 25th, median, 75th, and 95th percentiles. Apart from that, there are several other interesting insights that I managed to dig up, to find out more you can pressing the view project button below.

Dota 2 Competitive Picks

In this project, I explored the Dota 2 Competitive Picks dataset that I obtained through the Kaggle site. From this dataset, I learned to process data using Python, starting from importing data, checking and treating null values, checking and treating duplicate data, applying aggregation functions as part of data exploration, creating graphs, and so on.

When the Dota 2 Competitive dataset was processed, I found some interesting insights from the dataset. First, there is a relationship between the roles each hero has on win rate, time picked and times banned, heroes with roles disabler have greater win rates and times picked compared to other roles, while heroes with the biggest times banned held by a hero with an escape role because perhaps this hero tends to be difficult to fight because of his agile movement abilities. Apart from that, there are other interesting insights that might be useful or a reference for Dota 2 players, such as a comparison of the winning percentage of niche heroes with meta heroes, or the versatility of heroes' abilities based on their primary attributes.
